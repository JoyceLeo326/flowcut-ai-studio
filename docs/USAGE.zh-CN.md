# VisionCut AI 使用说明

VisionCut AI 是意图驱动、本地优先的视频创造系统：你描述目标，系统先生成可审阅的剪辑方案，再由本地步骤或可选的 ChatCut 云端流程执行。

## 1. 启动

在 PowerShell 中运行：

```powershell
cd "D:\整理后的文件_2026-05-24\04_比赛活动资料\workflow\FlowCut-AI-Studio"
.\start-web.ps1
```

浏览器打开 `http://localhost:3200`。如果这个窗口关闭，服务也会停止；再次运行脚本即可恢复。

## 2. 从意图开始

在创作中心直接描述成片目标，例如：

> 把一段十分钟创业者口播剪成 60 秒竖屏，删除停顿和口头禅，使用自然字幕和少量产品补镜。

点击“开始设计”后，VisionCut 会创建项目并推荐最匹配的创作配方。也可以先点“口播精剪”“长转短”“人物故事”等入口，再补充自己的要求。

## 3. 导入素材

进入编辑器后点击“素材”工作面中的“导入”，可以一次选择多个视频、图片和音频。浏览器只读取你主动选择的文件，不会扫描整个 D 盘；素材和项目默认保存在当前浏览器。

## 4. 选择工作流

- **AI 配方**：选择口播精剪、长转短、播客导播、活动回顾、产品故事、教程、旅行、赛事、广告或预告等流程。
- **故事图**：用 Opening、Character、Conflict、Emotion、Climax、Ending 节点规划叙事；当前是可编辑故事草案，不是已经完成的素材分析。
- **视觉实验室**：浏览已完成的原创镜头，或建立概念图生成队列。
- **开放素材**：搜索 Openverse 可商用许可图片；导入前仍需核对并保留具体许可和署名。

“指导模式”面向第一次剪辑的用户，只展示结果和下一步；“专业模式”会展开停顿阈值、切点边距、B-roll、字幕、推近、响度和交付版本等参数。

## 5. 三种执行方式

- **本地**：只生成并执行本地可完成的步骤，素材不会上传。
- **混合**：画幅、顺排、片段收紧在本地执行；字幕、静音检测和语义精选生成 ChatCut 交接任务。
- **ChatCut**：方案主要面向 ChatCut 云端执行，适合需要语音转录或按内容选取高光的任务。

## 6. 生成与审阅方案

选择配方后点击“用这套流程开始”，再打开导演蓝图。每一步都有执行位置、状态和风险提示。确认后：

- 点击“应用本地步骤”修改时间线；旁边的撤销按钮会一次撤销整组修改。
- 点击“复制 ChatCut 任务”获得包含项目、素材文件名和云端步骤的标准交接包。
- 将任务粘贴到已经启用 ChatCut 插件的 Codex 任务中，再按提示附上对应视频。

VisionCut AI 不会因为生成交接包而自动上传视频。真正上传到 ChatCut 前应再次确认；相关云端操作可能消耗 ChatCut 积分。界面中的制作角色目前是工作流预览，不表示云端 Agent 已经分析素材。

## 7. 手工精修

- 左侧：素材、声音、文字、贴纸、特效、转场、字幕和调整工具。
- 中间：视频预览，可拖拽和缩放当前元素。
- 下方：多轨时间线，可剪切、移动、调速、静音和添加关键帧。
- 右侧“属性”：当前元素的位置、大小、音量、速度、混合、遮罩和特效。

## 8. 导出

点击右上角“导出”，选择 MP4 或 WebM、质量和是否包含音频，然后确认导出。浏览器默认下载目录可能在 C 盘；如果要严格保持 C 盘零媒体占用，请在浏览器设置中把下载目录改为 D 盘，或每次导出后立即移动文件。

## 9. 手机和平板

底部固定提供“AI、素材、预览、时间线”四个工作面。手机适合表达意图、导入、审阅、预览和导出；平板可完成更完整的 AI 流程。复杂多轨精修仍建议使用桌面端。

## 10. 当前限制

- 浏览器版不能直接读取任意本地路径，必须由用户选择文件。
- ChatCut 与 VisionCut AI 目前通过可审阅的 JSON 交接包和成片回传连接，不是实时双向时间线同步。
- 本地“收紧片段首尾”是确定性规则；真正的静音检测、语义精选和语音字幕由 ChatCut 执行。
- 真实场景识别、人物/物体理解、情绪推断、留存预测、多 Agent 编排、Creator DNA 和云端渲染仍属于生产架构路线，不应把当前规则规划器理解为这些能力已经上线。
